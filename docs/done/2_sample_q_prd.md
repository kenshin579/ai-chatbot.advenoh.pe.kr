# PRD: AI Chatbot 예시 질문 & 입력 UI 개선

## 1. 배경

현재 AI Chatbot(ai-chatbot.advenoh.pe.kr)의 초기 화면은 단순한 안내 텍스트만 표시한다.
Google Scholar 실험실처럼 **예시 질문 카드**를 제공하면 사용자가 챗봇의 용도를 직관적으로 이해하고, 클릭 한 번으로 대화를 시작할 수 있다.

### 참고 이미지

Google Scholar 실험실 UI:
- 화면 중앙에 아이콘 + 안내 메시지
- 하단에 예시 질문 카드 리스트 (클릭 시 자동 입력)
- 입력창이 하단 중앙에 고정, 전송 버튼(▶) 포함

## 2. 목표

1. 초기 빈 화면에 **블로그별 예시 질문 카드**를 표시하여 사용자 진입 장벽 낮추기
2. 입력 UI를 Google Scholar 스타일로 개선 (둥근 입력창 + 전송 아이콘)

## 3. 적용 범위

| 파일 | 설명 |
|------|------|
| `frontend/src/constants/sampleQuestions.ts` | 블로그별 예시 질문 상수 (신규) |
| `frontend/src/components/MessageList.tsx` | 빈 화면 → 예시 질문 카드 UI |
| `frontend/src/components/ChatInput.tsx` | 입력창 UI 개선 |
| `frontend/src/components/ChatWindow.tsx` | 예시 질문 클릭 핸들러 연결 |

## 4. 기능 상세

### 4.1 예시 질문 카드

#### 현재 상태
- `"블로그에 대해 질문해 보세요!"` 텍스트만 표시 (`MessageList.tsx:55-60`)

#### 변경 사항

**초기 빈 화면 레이아웃:**
```
┌─────────────────────────────────────┐
│                                     │
│         🤖 아이콘 + 안내 메시지       │
│                                     │
│   ┌───────────────────────────────┐ │
│   │ 예시 질문 1                    │ │
│   └───────────────────────────────┘ │
│   ┌───────────────────────────────┐ │
│   │ 예시 질문 2                    │ │
│   └───────────────────────────────┘ │
│   ┌───────────────────────────────┐ │
│   │ 예시 질문 3                    │ │
│   └───────────────────────────────┘ │
│   ┌───────────────────────────────┐ │
│   │ 예시 질문 4                    │ │
│   └───────────────────────────────┘ │
│                                     │
│   ┌─────────────────────────────▶┐  │
│   │ 블로그에 질문하기              │  │
│   └──────────────────────────────┘  │
└─────────────────────────────────────┘
```

**동작:**
- 예시 질문 카드를 클릭하면 해당 질문이 자동으로 전송됨 (입력창에 넣는 게 아닌 바로 전송)
- 대화가 시작되면 예시 질문 카드는 사라지고 메시지 리스트로 전환

**블로그별 예시 질문 (예시):**

| blog_id | 예시 질문 |
|---------|---------|
| `blog-v2` | "Go 관련 글이 있나요?", "Spring Boot 시리즈 추천해줘", "최근 작성된 글 알려줘", "Docker 관련 글 요약해줘" |
| `investment` | "배당주 관련 글이 있나요?", "ETF 투자 전략에 대해 알려줘", "최근 시장 분석 글 있어?", "포트폴리오 관련 글 추천해줘" |
| `inspireme` | "오늘의 명언 추천해줘", "도전과 용기에 관한 명언 알려줘", "스티브 잡스 명언이 있나요?", "힘이 되는 짧은 명언 추천해줘" |

### 4.2 입력창 UI 개선

#### 현재 상태
- `<Input>` + `<Button>전송</Button>` (좌우 배치, border-t 구분선) (`ChatInput.tsx`)

#### 변경 사항

Google Scholar 스타일로 개선:

1. **입력창 형태**: 둥근 모서리(`rounded-full` or `rounded-2xl`), 충분한 패딩
2. **전송 버튼**: 입력창 내부 오른쪽에 위치하는 아이콘 버튼 (▶ 또는 Send 아이콘)
3. **placeholder**: `"블로그에 질문하기"` (간결하게)
4. **위치**: 하단 중앙 고정, 좌우 여백 있음 (max-width 제한)
5. **그림자/테두리**: 부드러운 shadow로 입력창 강조

```
변경 전: [___입력창___] [전송]

변경 후: ┌────────────────────────────▶┐
         │  블로그에 질문하기            │
         └─────────────────────────────┘
```

## 5. 논의 사항

1. **예시 질문 개수**: 3~4개가 적당한지?
2. **예시 질문 클릭 동작**: 클릭 시 바로 전송 vs 입력창에 텍스트만 채우기 (사용자가 수정 가능)
3. **예시 질문 랜덤화**: 매번 같은 질문을 보여줄지, 풀에서 랜덤으로 뽑을지
4. **예시 질문 데이터 소스**: 프론트엔드 상수 vs 백엔드 API로 제공 (콘텐츠 변경 시 배포 없이 업데이트 가능)
6. **입력창 textarea 전환**: 현재 `<Input>`(한 줄)인데, 긴 질문을 위해 `<Textarea>`로 변경할지 (auto-resize)
7. **다크모드 대응**: 예시 질문 카드 색상이 라이트/다크 모드에서 모두 잘 보이는지 확인 필요
